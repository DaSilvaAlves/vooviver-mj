## Story

**As a** potencial cliente da Maria Serqueira,
**I want** poder pedir um orçamento através de um formulário simples em 3 passos, ver respostas às minhas dúvidas frequentes, encontrar informação legal no rodapé e ter a certeza de que a página aparece correctamente nas redes sociais e no Google,
**so that** eu possa contactar a Maria com confiança e a página possa ser encontrada organicamente.

**Depends on:** Story 1.1 (projecto configurado), Story 1.2 (secções estáticas integradas em index.astro)

## Acceptance Criteria

1. `ContactForm.astro` implementado com formulário multi-step em 3 passos (JavaScript vanilla com `<script>` tag nativo do Astro — sem Astro Island, sem framework adicional):
   - **Passo 1:** Nome (obrigatório), Email (obrigatório), Telefone (obrigatório) — 3 campos
   - **Passo 2:** Destino (dropdown com 4 opções + "Outro"), Datas aproximadas (campo texto), Número de pessoas (número, min 1)
   - **Passo 3:** Observações livres (textarea, opcional), "Como conheceu a Maria?" (dropdown: Instagram, WhatsApp, Google, Referência, Outro)
2. Barra de progresso visual com 3 etapas numeradas ("1 · 2 · 3") visível durante todo o preenchimento; etapa activa destacada.
3. Submissão parcial no Passo 1: ao avançar do Passo 1 para o Passo 2, os dados do Passo 1 são submetidos para Formspree (lead capturado mesmo com abandono posterior).
4. Alternativa WhatsApp sempre visível ao lado do formulário: `<a href="https://wa.me/351000000000">Prefere falar pelo WhatsApp?</a>` — link abre em nova tab.
5. Validação inline sem reload: campos obrigatórios validados antes de avançar para o próximo passo; mensagens de erro visíveis inline junto ao campo.
6. Submissão final do Passo 3 para Formspree (endpoint configurado em variável de ambiente `FORMSPREE_ENDPOINT`); em caso de erro de rede, mostrar mensagem "Erro ao enviar. Tente novamente ou contacte pelo WhatsApp."
7. Confirmação visual após envio bem-sucedido: mensagem "Obrigado! A Maria responde em 24 horas." com ícone de sucesso; formulário substituído pela mensagem (sem reload de página).
8. Checkbox RGPD antes do botão de submissão final: "Aceito a Política de Privacidade e o tratamento dos meus dados para receber o orçamento solicitado." — obrigatório para submeter.
9. Formulário acessível: `<label>` para cada input, atributos `required` e `aria-required="true"` nos campos obrigatórios, `aria-describedby` para mensagens de erro.
10. Secção do formulário tem `id="formulario"` para que os CTAs da Story 1.2 possam fazer scroll até aqui.
11. `FAQ.astro` implementado: lista de 6 perguntas lidas de `src/data/faq.json`, accordion/collapse implementado em CSS puro usando `<details>` e `<summary>` HTML nativos (zero JavaScript externo), cada FAQ tem `id` único para deep linking.
12. Schema markup FAQPage injectado como `<script type="application/ld+json">` em `src/layouts/Layout.astro` (condicionalmente apenas quando há FAQs) ou directamente em `FAQ.astro`.
13. `Footer.astro` implementado com todos os elementos do PRD FR-09: RNAVT N.12366, link RGPD/Política de Privacidade (URL placeholder), métodos de pagamento aceites (ícones ou texto), Livro de Reclamações (link para `https://www.livroreclamacoes.pt`), link discreto "Queres ser consultor VooViver?" (separado visualmente dos links principais), copyright "© 2026 Maria Serqueira — Consultora VooViver".
14. Meta tags SEO implementadas em `Layout.astro` via props: `<title>`, `<meta name="description">`, Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`, `og:type`), Twitter Cards (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`).
15. Valores SEO para a landing page da Maria (PRD FR-10): Title = "Maria Serqueira — Viagens de Grupo desde 870 EUR | VooViver", meta description optimizada com CTA.
16. Schema markup TravelAgency + TouristTrip injectado em `<script type="application/ld+json">` em `index.astro` ou `Layout.astro`.
17. `<link rel="canonical">` presente em `Layout.astro`.
18. `npm run build` executa sem erros com todas as secções integradas.

## Dev Notes

### Formspree — Configuração

**Formspree é gratuito até 50 submissões/mês.** Para criar endpoint:
1. Registar em formspree.io
2. Criar novo formulário — obter endpoint `https://formspree.io/f/XXXXXXXX`
3. Adicionar ao `.env` local: `PUBLIC_FORMSPREE_ENDPOINT=https://formspree.io/f/XXXXXXXX`
4. Adicionar ao Vercel Dashboard → Project Settings → Environment Variables: `PUBLIC_FORMSPREE_ENDPOINT`

**CRÍTICO:** Prefixo `PUBLIC_` é obrigatório para Astro expor variáveis de ambiente ao cliente.

```astro
---
const endpoint = import.meta.env.PUBLIC_FORMSPREE_ENDPOINT;
---
```

Se o endpoint não estiver configurado no momento do desenvolvimento, usar `https://formspree.io/f/PLACEHOLDER` como fallback visível — o formulário mostrará erro de rede ao submeter, o que é aceitável para o MVP.

### Formulário Multi-Step — Implementação Recomendada (Zero Dependencies)

Dado que Astro é SSG com zero JS por defeito, a forma mais simples é usar JavaScript vanilla dentro de um `<script>` no componente:

```astro
---
// ContactForm.astro — sem framework adicional
---
<div id="contact-form-container">
  <div id="step-1" class="form-step active">...</div>
  <div id="step-2" class="form-step hidden">...</div>
  <div id="step-3" class="form-step hidden">...</div>
  <div id="success-msg" class="hidden">...</div>
</div>

<script>
  // Gestão de passos
  let currentStep = 1;
  const steps = document.querySelectorAll('.form-step');

  function showStep(n) {
    steps.forEach((s, i) => s.classList.toggle('hidden', i !== n - 1));
    currentStep = n;
    updateProgress();
  }
  // ... validação, fetch, etc.
</script>
```

Esta abordagem NÃO requer `client:load` — o script é carregado normalmente pelo browser.

[Source: PRD-LANDING-PAGE-MARIA-SERQUEIRA.md#fr-07-formulario-multi-step-progressive-disclosure]

### RGPD — Requisito Obrigatório (NFR-06)

O formulário DEVE ter checkbox de consentimento antes da submissão:

```html
<label>
  <input type="checkbox" id="rgpd-consent" required aria-required="true" />
  Aceito a <a href="/politica-de-privacidade">Política de Privacidade</a> e o tratamento dos meus dados para receber o orçamento solicitado.
</label>
```

- A submissão deve ser BLOQUEADA se o checkbox não estiver marcado
- Não enviar dados para Formspree sem consentimento

[Source: PRD-LANDING-PAGE-MARIA-SERQUEIRA.md#5-requisitos-nao-funcionais - NFR-06]

### FAQ — HTML `<details>` Nativo

`<details>` e `<summary>` são suportados em todos os browsers modernos sem JavaScript. Evitar qualquer biblioteca de accordion externa.

CSS básico:
```css
details summary {
  cursor: pointer;
  font-weight: 600;
  padding: 1rem 0;
  border-bottom: 1px solid #e2e8f0;
  list-style: none; /* remove seta default em alguns browsers */
}
details[open] summary { color: var(--color-primary); }
details p { padding: 0.75rem 0 1rem; color: var(--color-text-secondary); }
```

[Source: PRD-LANDING-PAGE-MARIA-SERQUEIRA.md#fr-08-faq-com-schema]
