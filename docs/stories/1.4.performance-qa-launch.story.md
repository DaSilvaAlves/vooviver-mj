## Story

**As a** Maria Serqueira e visitante da landing page,
**I want** uma página com Lighthouse Performance >= 95, zero erros de consola, acessibilidade WCAG 2.1 AA validada, schema markup correcto e URL público estável no Vercel,
**so that** a landing page possa ser partilhada e usada em campanhas reais com confiança na sua qualidade e performance.

**Depends on:** Story 1.1 (setup), Story 1.2 (secções estáticas), Story 1.3 (formulário, FAQ, footer, SEO)

## Acceptance Criteria

1. Lighthouse Performance score >= 95 medido no URL de produção Vercel (não em localhost).
2. LCP (Largest Contentful Paint) < 1.5s na conexão "Fast 3G" simulada.
3. CLS (Cumulative Layout Shift) < 0.1.
4. FID/INP (Interaction to Next Paint) < 100ms.
5. Lighthouse Accessibility score >= 90.
6. Lighthouse SEO score >= 95.
7. Lighthouse Best Practices score >= 90.
8. Todas as imagens servidas em formato WebP ou AVIF (convertidas via `<Image>` do Astro ou pré-convertidas manualmente).
9. Imagem hero (above-the-fold) tem `loading="eager"` e `fetchpriority="high"`; todas as outras imagens têm `loading="lazy"`.
10. `sitemap.xml` gerado automaticamente via integração `@astrojs/sitemap` e acessível em `/sitemap-index.xml` ou `/sitemap.xml`.
11. `robots.txt` referencia o sitemap: `Sitemap: https://{dominio-final}/sitemap.xml`.
12. Zero erros na consola do browser (DevTools → Console) na URL de produção.
13. Zero erros de rede 404 para assets (imagens, CSS, JS) na URL de produção.
14. Google Rich Results Test valida FAQPage schema sem erros (aviso é aceitável).
15. Facebook Debugger mostra OG tags correctas (título, descrição, imagem).
16. Formulário funcional em produção: submissão de teste chega ao Formspree dashboard.
17. Responsividade verificada em: iPhone SE (375px), iPhone 14 Pro (390px), iPad (768px), desktop (1440px) — sem scroll horizontal em nenhum viewport.
18. Deploy estável no Vercel: URL final configurado e acessível (pode ser subdomínio `vooviver-mj.vercel.app` ou domínio personalizado se disponível).

## Dev Notes

### Lighthouse — Targets e Estratégias de Optimização

**Target obrigatório (PRD NFR-01):** Performance >= 95 em produção.

**Principais causas de score baixo e soluções:**

| Issue Comum | Solução Astro |
|-------------|---------------|
| Imagens pesadas | `<Image>` com `width` e `height` definidos + WebP auto |
| Font loading | `font-display: swap` + `<link rel="preconnect">` |
| Render-blocking CSS | Tailwind gera CSS mínimo em build — normalmente OK |
| LCP alto | Hero image com `fetchpriority="high"` e `loading="eager"` |
| CLS por imagens | Sempre especificar `width` e `height` nos `<img>`/`<Image>` |
| CLS por fontes | `font-display: swap` evita FOUT mas pode causar CLS ligeiro |

**Astro já optimiza automaticamente:**
- Code splitting (zero JS por defeito em SSG)
- CSS tree-shaking (Tailwind remove classes não usadas em build)
- HTML minification

[Source: PRD-LANDING-PAGE-MARIA-SERQUEIRA.md#5-requisitos-nao-funcionais - NFR-01, NFR-04]

### Astro Sitemap — Configuração

```js
// astro.config.mjs
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';
import sitemap from '@astrojs/sitemap';

export default defineConfig({
  site: 'https://vooviver-mj.vercel.app', // substituir pelo domínio final
  output: 'static',
  integrations: [sitemap()],
  vite: {
    plugins: [tailwindcss()],
  },
});
```

**CRÍTICO:** O campo `site` é obrigatório para o sitemap ser gerado com URLs absolutas.

### Fontes Self-Hosted vs Google Fonts

**Recomendação para performance máxima:** usar `@fontsource` (self-hosted):
```bash
npm install @fontsource/inter @fontsource-variable/inter @fontsource/jetbrains-mono
```

Em `global.css`:
```css
@import "@fontsource-variable/inter";
@import "@fontsource/jetbrains-mono/700.css";
```

Vantagem: elimina requisição externa ao Google Fonts → melhoria de ~0.2-0.5s no LCP.
